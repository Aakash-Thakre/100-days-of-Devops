# Fix Python App Deployed on Kubernetes Cluster

One of the DevOps engineers was trying to deploy a python app on Kubernetes cluster. Unfortunately, due to some mis-configuration, the application is not coming up. Please take a look into it and fix the issues. Application should be accessible on the specified nodePort.

- The deployment name is `python-deployment-devops`, its using `poroko/flask-demo-appimage`. The deployment and service of this app is already deployed.
- nodePort should be `32345` and `targetPort` should be python flask app's default port.

> Note: The kubectl on jump_host has been configured to work with the kubernetes cluster.

## Steps

1. Let's see the pod status

    ```sh
    kubectl get pods
    kubectl describe pod pod-name
    ```
    It shows ImagePullError 

    ```
    > `docker.io/poroko/flask-app-demo:latest` the dpeloyment using wrong docker image

    correct image name is `poroko/flask-demo-appimage

2. Let's create a `deployment.yaml` file from existing deployment

    ```sh
    kubectl get deployments.apps python-deployment-devops -o yaml > deployment.yaml
    ```

3. Correct the docker image

    ```sh
    vi deployment.yaml
    ```

    > replace docker image in container section with accurate image

4. Apply the changes to the deployment

    ```sh
    kubectl apply -f deployment.yaml
    kubectl get pods # container runs successfully 
    ```

    > tried to access app, but its not working.

5. Now let's check the service status

    ```sh
    kubectl get svc
    ```
    > service is already running but it's mapped with wrong target port

6. Fixing service

    ```sh
    kubectl get svc python-service-devops -o yaml > service.yaml
    ```

    > change the target port in `service.yaml` file to correct port mentioned 32345

    ```sh
    vi service.yaml
    ```

    > configure service

    ```sh
    kubectl apply -f service.yaml
    ```

7. Verify results

   ```sh
   kubecytl get all
   ```
