## Scenario

The Nautilus application development team has prepared the beta version of a Java-based application and plans to deploy it on **App Server 3** in Stratos Datacenter using **Apache Tomcat**.

The requirement is to:

- Install Tomcat on App Server 3
- Configure Tomcat to run on a custom port
- Deploy the application as the ROOT application so it loads directly from the base URL

---

## Objective

- Install Apache Tomcat on: `stapp03`
- Configure Tomcat to run on port: `5001`
- WAR file location (Jump Host): `/tmp/ROOT.war`
- Deploy location: `/var/lib/tomcat/webapps/`
- Application should be accessible at: http://stapp03:5001

  
---

## Concepts Used

- Apache Tomcat
- Java WAR deployment
- Linux services (systemctl)
- SCP file transfer
- Linux permissions and ownership
- Tomcat server.xml configuration

---

## STEP-BY-STEP COMPLETE SOLUTION
1. Go to App Server 3
```bash
ssh stapp03
```

2. Install Tomcat
``` bash
sudo yum install tomcat -y
```

3. Change Tomcat Port to 5001

Open config file:
```bash
sudo vi /etc/tomcat/server.xml
```

4. Find:

Connector port="8080"

Change to:

Connector port="5001"


Save and exit.

Start & Enable Tomcat

```bash
sudo systemctl start tomcat
sudo systemctl enable tomcat
```

2. Go to Jump Host and Copy ROOT.war

On Jump host:

```bash
scp /tmp/ROOT.war stapp03:/tmp/
```

6. On stapp03, Move ROOT.war to Tomcat

```bash
ssh stapp03
sudo mv /tmp/ROOT.war /var/lib/tomcat/webapps/
```

7. Fix Permissions

```bash
sudo chown tomcat:tomcat /var/lib/tomcat/webapps/ROOT.war
```

8. Restart Tomcat
```bash
sudo systemctl restart tomcat
```

9. Test
```bash
curl http://stapp03:5001
```
